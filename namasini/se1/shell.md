
## 커널
+ 하드웨어와 운영체제의 근간
+ 운영 체제(OS)의 심장이자 OS를 규정짓는 매우 중요한 부분. 하드웨어의 자원을 자원이 필요한 프로세스에 나눠주고, 덩달아 프로세스 제어(태스크 매니저), 메모리 제어, 프로그램이 운영체제에 요구하는 시스템 콜 등을 수행하는 부분으로 운영체제 맨 하부에서 돌아간다. 현재 많이 사용되고 있는 OS는 커널 위에 여러 가지 레이어를 올린 것. 이렇기 때문에 커널이 날아가게 되면 운영체제를 못쓰게 된다
+ 어쨌든 커널로 운영체제의 정체성을 결정하기에 매우 중요하다고 볼 수 있다. 페도라, 우분투 등이 다 리눅스로 묶이는 것도 이들이 같은 리눅스 커널을 사용하고 있기 때문.


# 리눅스
+ 커널의 일종인 리눅스 커널, 또는 리눅스 커널을 사용하는 운영 체제를 가리키는 말이기도 하다 
+ 컴퓨터 역사상 가장 많은 사람이 들어간 오픈 소스 프로젝트이며, 모바일 운영 체제로 유명한 안드로이드가 이것을 기반으로 한다

+ CLI vs GUI

## 리눅스의 종류 대표
### 레드헷계열
 레드햇 리눅스와, 그 커뮤니티 버전인 페도라에서 파생된 배포판들이다. 패키지 형식은 .rpm이며 패키지 관리자로 yum[61]을 사용하는 것이 특징이다. 서버용으로 사용되는 경우가 대부분이다. 얼마 없는 상용 리눅스 중에서도 굉장히 잘나가는 편인데, 이는 서버 시장이 주 타겟인 배포판이기 때문.

    + 레드헷
        + CentOS

### 데비안 계열
데비안에서 파생된 배포판들. 패키지 형식은 .deb이며, 패키지 관리자로 apt를 이용한다. 주로 개인용으로 사용되고, 일반 사용자를 주 타겟으로 하기 때문에 사용자 친화적인 배포판이 많다.


+ 우분투
    + 리눅스 민트
        + 하모니카[64]
    + 쿠분투
    + 주분투
+ 크롬 OS/크로뮴 OS용 리눅스 도구[68]
+ 스팀
  
### 안드로이드 계열
모바일 운영체제인 안드로이드 계열 배포판들. 패키지 형식은 apk이다. 기존 리눅스와는 구조가 크게 달라서 다른 배포판과 함께 엮지 않는 경우도 많지만, 어쨌거나 리눅스로 본다면 현재 일반 사용자를 대상으로 한 리눅스 배포판 중에는 가장 메이저하다. 구글의 PC용 OS인 크롬OS 역시 안드로이드 앱이 호환된다.

## 프로세스
+ 프로그램 내에서 실행되는 진행흐름
+ 컴퓨터는 동시다발적 활동이 가능한것같지만, 내부 심연에서는 프로세스의 순서대로 진행되고 있다.

## 쓰레드
+ 프로세스 내에서 실행되는 실행흐름
+ 한 프로세스는 여러개의 쓰레드를 가질 수 있다
+ 쓰레드의 실행 순서는 보장되지 않기때문에 어떤쓰레드가 먼저 실행될지는 알수없고, 이때문에 알고리즘 문제가 발생하고 해결법들이 나오고있다.


## Ipc
+ Inter Process Communication
+ 프로세스간 통신. 컴퓨터 운영체제에서 프로세스 간에 데이터를 주고받는 방식, 원래는 한 프로세스는 자신의 메모리만 가지고있다. 스레드를 이용하여 메모리공간을 공유한다.
+ 프로세스의 재사용성을 높이기 위해 필요하다
+ ex) 스마트폰 메세지,전화기앱,사진이있는데  특정한앱에서 사진첩을 열어주세요,메세지를 열어주세요 이런경우.   닫으면 다시 내 프로세스로 돌아는 경우


+ 메일슬롯(파일)
  + 단방향
+ 파이프
  + 유닉스 시스템에 많이쓰이는 IPC 구현중 하나
  + 유닉스 호환 커널은 파이프를 생성할수있다. 시스템콜(일종의 하드웨어 함수)
  + 셸에서 | 앞으로 프로그램 명이 나온다면 앞의프로그램과 뒤의 프로그램사이에 파이프가 생성되고, 앞의결과가 뒤의 입력으로 들어간다.
  + 앞의 프로세스의 결과를 뒤의 프로세스로 전달하는것
  
+ 명명되지않은 익명 파이프
  + os에서 제공하는 기본적인 파일기반 단방향 ipc

+ 소켓(*)
  + 두 네트워크 프로그램 사이에서 데이터의 입출력 관문
  + TCP연결이 수립되면 스트림소켓(스트리밍, 유튜브, 도착을 체크함) UDP를 사용하면 데이터그램 소켓(생방송, 상호작용이아닌 무조건 보냄)이라고 불린다.
    그래서 TCP가 비교적 느리나, 데이터손실은 데이터그램이 더 크다
  + 웹프로세스 2개를 연결한다는점에서 훌륭하다.
+ 시그널
+ 공유메모리

## 데몬화 절차
+ 프로세스는 자신을 생성한 프로세스(즉,부모 프로세스)에 종속되며, 부모 프로세스가 종료되면 같이 종료되는 특성이 있다.
+ 데몬이란 부모프로세스가 종료되어도 종료되지 않는 프로세스를 만드는 것을 의미한다.
+ 데몬화절차는 자식 프로세스의 부모를 root 프로세스로 교체하는것을 의미한다
